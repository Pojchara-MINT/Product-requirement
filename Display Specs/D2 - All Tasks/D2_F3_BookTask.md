# D2 — F3: จองงาน (Book Task)

> **Display:** D2 | **Function:** 3 of 3
> **Back to overview:** [D2_Overview.md](D2_Overview.md)

---

**Objective:** Checker / Signer จองงาน FTT เพื่อนำไปรีวิวบน D3 (งานของฉัน)

> **ปุ่มจองงานแสดงทุก row เสมอ** — User สามารถจองงานได้แม้ว่า สถานะงาน = ไม่พร้อมตรวจ

---

## Flow

| Step | Action | System Response |
|------|--------|----------------|
| 1 | User กดปุ่ม **"จองงาน"** บน FTT ที่ต้องการ | ระบบแสดง Confirmation Modal |
| 2 | User กดยืนยัน | FTT ถูกจองให้ User คนนี้ → FTT หายออกจาก Pool (ไม่แสดงใน D2 ของ User นี้อีก) → FTT ปรากฏใน **D3 งานของฉัน** |

---

## Use Cases

### Case 1: FTT ยังไม่มีผู้จอง (ผู้ตรวจ = `-`)

| | |
|--|--|
| **Modal Wording** | **"ต้องการจองงาน"** |
| **Buttons** | ยกเลิก (แดง) / ยืนยัน (เขียว) |
| **Expected Result (ยืนยัน)** | FTT ถูกจองให้ User — หายออกจาก D2 Pool — ปรากฏใน D3 งานของฉัน |
| **Expected Result (ยกเลิก)** | Modal ปิด — ไม่มีการเปลี่ยนแปลง |

### Case 2: FTT มีผู้จองอยู่แล้ว (ผู้ตรวจ = ชื่อ User อื่น)

| | |
|--|--|
| **Modal Wording** | **"งานของ {ชื่อผู้ตรวจเดิม} ต้องการจองงานแทนหรือไม่"** |
| **Buttons** | ยกเลิก (แดง) / ยืนยัน (เขียว) |
| **Expected Result (ยืนยัน)** | FTT ถูกโอนมาจองให้ User คนใหม่ — หายออกจาก D2 ของ User คนใหม่ — ปรากฏใน D3 งานของฉัน ของ User คนใหม่ |
| **Expected Result (ยกเลิก)** | Modal ปิด — ผู้จองเดิมยังคงเป็นเจ้าของงาน |

---

## Display Rule After Booking

> FTT ที่ User จองแล้วจะ**ไม่ปรากฏ**ใน D2 ของ User คนนั้น
> เหตุผล: D2 แสดงเฉพาะงานที่ยังรอการจองจาก User คนนี้เท่านั้น
> FTT ที่จองแล้วสามารถดูได้ใน **D3 — งานของฉัน**
