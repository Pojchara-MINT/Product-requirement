# D4 — F2: Transaction Detail

> **Display:** D4 | **Function:** 2 of 2
> **Back to overview:** [D4_Overview.md](D4_Overview.md)

---

**Objective:** แสดงรายละเอียด FTT แบบ Read-only พร้อม State History Timeline และเอกสารประกอบ — ให้ User ติดตามสถานะรายการแบบครบถ้วน

---

## Entry Point

> User คลิก Row ใน Transaction History List (D4_F1) → ระบบเปิด Transaction Detail (L2) ของ FTT นั้น

---

## Page Layout

D4_F2 ใช้ **3-Panel Layout** เช่นเดียวกับ D3:

| Panel | ตำแหน่ง | ชื่อ | หน้าที่ |
|-------|---------|------|---------|
| Panel 1 | ซ้าย | State History | แสดง Timeline การเปลี่ยนสถานะของ FTT |
| Panel 2 | กลาง | Document Viewer | แสดงเอกสารประกอบของ FTT |
| Panel 3 | ขวา | FTT Detail | แสดงรายละเอียด FTT (Read-only) |

---

## Panel 1 — State History (ซ้าย)

### Header

| Element | Description |
|---------|-------------|
| สถานะปัจจุบัน | แสดงสถานะล่าสุดของ FTT — เช่น **"สถานะปัจจุบัน: สำเร็จ"** |

### State Cards

แสดง State Card ตามลำดับเวลา (จากเก่าสุดบนสุด ถึงล่าสุดล่างสุด)

| Field ใน Card | Description | Format |
|--------------|-------------|--------|
| ชื่อ State | ชื่อขั้นตอน เช่น สร้างรายการ / รอผู้ตรวจสอบ / รออนุมัติ / รอผลการโอนเงิน | Text |
| ผลลัพธ์ | ผลของขั้นตอนนั้น เช่น สร้างรายการโอนเงินสำเร็จ / ผู้ตรวจสอบอนุมัติ / อนุมัติรายการ / ผลโอนเงินสำเร็จ | Link / Text |
| ชื่อ-สกุล | ชื่อผู้ดำเนินการในขั้นตอนนั้น | Text |
| วันเวลา | Timestamp ที่ดำเนินการ | DD MMM YY \| HH:MM น. |
| ผู้ทำรายการ | ชื่อ User หรือ System ที่ดำเนินการ | Text |

### State Card Sequence (ตัวอย่าง)

| ลำดับ | ชื่อ State | ผู้ทำรายการ |
|-------|-----------|-------------|
| 1 | สร้างรายการ | Maker / QA from LOS |
| 2 | รอผู้ตรวจสอบ | Checker |
| 3 | รออนุมัติ | Signer |
| 4 | รอผลการโอนเงิน | Bank System |

> จำนวน Card ขึ้นกับสถานะจริงของ FTT — แสดงเฉพาะ State ที่เกิดขึ้นแล้ว

---

## Panel 2 — Document Viewer (กลาง)

แสดงเอกสารประกอบของ FTT — ใช้ Tabs เดียวกับ D3:

| Tab | เอกสาร |
|-----|--------|
| 1 | สมุดบัญชี |
| 2 | บัตรประชาชน |
| 3 | ใบเปลี่ยนชื่อ-นามสกุล |
| 4 | ทะเบียนสมรส |
| 5 | แบบสัมภาษณ์ |
| 6 | รูปอื่นๆ |

> แสดง Tab ทั้งหมดเสมอ — หากไม่มีเอกสารในบาง Tab ให้แสดงสถานะ "ไม่มีเอกสาร" หรือ placeholder

---

## Panel 3 — FTT Detail (ขวา)

แสดงรายละเอียด FTT แบบ **Read-only** — ไม่มี Action Buttons ใดๆ

### ส่วนที่ 1 — ข้อมูลรายการ

| Field | Description |
|-------|-------------|
| เลขที่อ้างอิง | Reference number + Badge (System/Manual) + Badge Online (ถ้า channel = Online) |
| รหัสรายการ | Internal transaction ID |
| ประเภทงาน | สินเชื่อ / คืนเงิน |
| เลขบัตรประชาชน | เลขบัตรของผู้รับโอน |
| ประเภทสินเชื่อ | ประเภทสินเชื่อที่เกี่ยวข้อง |
| หลักประกัน | ข้อมูลหลักประกัน |
| ชื่อ-สกุล (ภาษาไทย) | ชื่อผู้รับโอน + ตัวบ่งชี้ความตรง |
| ชื่อ-สกุล (ภาษาอังกฤษ) | ชื่อผู้รับโอน (EN) + ตัวบ่งชี้ความตรง |
| ช่องทาง | Online / Branch |
| เลขที่บัญชี / พร้อมเพย์ | บัญชีปลายทาง |
| จำนวนเงินสุทธิ | ยอดโอน (บาท) |

### ส่วนที่ 2 — ผลตรวจสอบและสถานะ

| Field | Description | Badge |
|-------|-------------|-------|
| ผลตรวจสอบบัญชีปลายทาง | ผลการ Verify บัญชีจากธนาคาร | ผ่าน (เขียว) / ไม่ผ่าน (แดง) / ยังไม่ได้รับผล (เทา) |
| ผลเงื่อนไขโอนเงิน | ผล Internal Rule | ผ่าน (เขียว) / ไม่ผ่าน (แดง) / ยังไม่ได้รับผล (เทา) |
| สถานะรายการ | สถานะปัจจุบันของ FTT | สำเร็จ (เขียว) / ปฏิเสธ (แดง) / ยกเลิก (แดง) / รอดำเนินการ (เหลือง) |
| ผลการโอนเงินจากธนาคาร | ผลตอบกลับจากธนาคาร | Success (เขียว) / Fail (แดง) / Pending (เหลือง) |

> ทุก Field ใน Panel 3 เป็น Read-only — ไม่มีปุ่ม ยืนยัน / ยกเลิก

## Display Difference vs D3 Panel 3

| Element | D3 Panel 3 (My Tasks) | D4_F2 Panel 3 (Transaction Detail) |
|---------|-----------------------|-------------------------------------|
| ปุ่ม ยืนยัน | แสดง | **ไม่แสดง** |
| ปุ่ม ยกเลิก | แสดง | **ไม่แสดง** |
| Rejection Form | แสดงเมื่อกดยกเลิก | **ไม่แสดง** |
| หมายเหตุ (Action) | แสดง (Optional) | **ไม่แสดง** |
| ผลตรวจสอบและสถานะ | แสดงใน D3_F4 | แสดงใน Panel 3 ส่วนที่ 2 |
| State History | ไม่แสดงใน Panel 3 | แสดงใน Panel 1 (Timeline) |
